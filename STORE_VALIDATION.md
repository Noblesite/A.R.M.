## Preconditions

- Install the debug APK generated by `./gradlew assembleDebug`.
- Confirm the device has no active A.R.M. test data that must be preserved.
- Confirm file manager access to Downloads.
- Confirm Cisco AnyConnect is not installed for the baseline run.

## Validation Procedure

1. Install A.R.M.
2. Launch A.R.M. and confirm the package is `net.noblesite.arm`.
3. Confirm the launcher label is A.R.M.
4. Confirm the first screen shows A.R.M. and Android Resource Monitor.
5. Tap Start.
6. Confirm Start disables, Stop enables, and a start snapshot appears.
7. Close and reopen A.R.M. while the session is running.
8. Confirm the active session and start snapshot are restored.
9. Tap Stop.
10. Confirm the latest completed session appears with start snapshot, stop snapshot, and session change details.
11. Confirm the generated summary includes duration, available-memory change, battery change, network RX/TX change, and low-memory observation.
12. Tap Export Report.
13. Confirm an export success message appears.
14. Open Downloads/A.R.M.
15. Confirm `arm-session-<session id>.txt` exists.
16. Open the report and confirm it includes the generated summary, start snapshot, stop snapshot, and session change details.
17. Install Cisco AnyConnect.
18. Repeat steps 5-16 for the post-install session.
19. Compare the baseline and post-install text reports.

## Pass Criteria

- The app installs and launches on each Android 11-14 target.
- Session start/stop controls enforce one active session at a time.
- Resource snapshots display without app crash.
- Active and latest completed sessions survive app restart.
- Text report export succeeds without external tooling.
- Baseline and post-install reports can be compared by a store engineer.

## Failure Capture

For any failure, record:

- Android version.
- Device model.
- A.R.M. APK version.
- Step number.
- Expected result.
- Actual result.
- Whether Cisco AnyConnect was installed.
- Screenshot or copied error text when available.
